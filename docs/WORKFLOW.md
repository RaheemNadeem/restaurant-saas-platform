# Delivery Workflow (Free Stack)

## Board
GitHub Project: Restaurant SaaS MVP Board

## Status Policy
- **Todo**: scoped and ready
- **In Progress**: assigned and actively worked
- **Done**: merged to `main` and validated

## Branching
- `main` protected baseline
- feature branches: `feat/<issue>-<short-name>`
- bug branches: `fix/<issue>-<short-name>`

## PR Rules
- Every PR must reference an issue: `Closes #<issue-number>`
- At least one review before merge
- CI must pass

## Sprint Cadence
- Monday: sprint plan
- Wednesday: status/risk check
- Friday: demo + KPI review
